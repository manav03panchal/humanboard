## Purpose and Scope

This document describes Zed's Item system, which provides a unified framework for displaying different types of content within workspace panes. The Item system defines how editors, search results, terminals, and other tools integrate into the workspace, including their lifecycle from creation through closure, serialization for persistence, and registration with the workspace.

For information about how panes organize items, see page 4.3 (Pane Management). For details on workspace organization and panel systems, see page 4.1 (Workspace Organization).

* * *

## Item System Overview

The Item system is Zed's abstraction for any content that can be displayed in a pane. Items are managed through a trait-based architecture that supports features like tab display, saving, navigation, collaboration, and persistence.


**Sources:** [crates/workspace/src/item.rs165-348](<https://github.com/zed-industries/zed/blob/4109c9dd/crates/workspace/src/item.rs#L165-L348>) [crates/workspace/src/workspace.rs612-903](<https://github.com/zed-industries/zed/blob/4109c9dd/crates/workspace/src/workspace.rs#L612-L903>) [crates/editor/src/items.rs1-100](<https://github.com/zed-industries/zed/blob/4109c9dd/crates/editor/src/items.rs#L1-L100>)

## The Item Trait

The `Item` trait is the fundamental interface that all pane content must implement. It defines how content appears in tabs, handles lifecycle events, and integrates with workspace features.

### Core Item Methods

Method Category| Key Methods| Purpose  
---|---|---  
**Tab Display**| `tab_content()`, `tab_content_text()`, `tab_icon()`, `tab_tooltip_text()`| Define tab appearance and tooltips  
**File Operations**| `can_save()`, `save()`, `save_as()`, `reload()`, `is_dirty()`, `has_conflict()`| Handle file persistence  
**Navigation**| `navigate()`, `set_nav_history()`, `breadcrumbs()`, `breadcrumb_location()`| Support navigation history and breadcrumbs  
**Lifecycle**| `deactivated()`, `workspace_deactivated()`, `on_removed()`, `added_to_workspace()`| Respond to lifecycle events  
**State**| `act_as_type()`, `for_each_project_item()`, `buffer_kind()`| Query item state and type  
**Features**| `can_split()`, `clone_on_split()`, `as_searchable()`| Enable optional features

## Specialized Item Traits

Beyond the core `Item` trait, several specialized traits provide additional functionality for specific item types.

### FollowableItem - Collaboration Support

The `FollowableItem` trait enables items to participate in real-time collaboration, where one user can follow another's navigation and edits. Key methods include:

  * `remote_id()`: Returns the collaboration identifier
  * `to_state_proto()` / `from_state_proto()`: Serialize/deserialize state for collaboration
  * `set_leader_id()`: Track who is being followed
  * `dedup()`: Prevent duplicate tabs when following


**Sources:** [crates/workspace/src/item.rs551-712](<https://github.com/zed-industries/zed/blob/4109c9dd/crates/workspace/src/item.rs#L551-L712>) [crates/editor/src/items.rs64-382](<https://github.com/zed-industries/zed/blob/4109c9dd/crates/editor/src/items.rs#L64-L382>)

### SearchableItem - Find and Replace

The `SearchableItem` trait provides find-and-replace functionality. Items implement `find_matches()` to search their content and `replace()` / `replace_all()` to perform replacements. The `BufferSearchBar` toolbar item coordinates search operations.

**Sources:** [crates/workspace/src/searchable.rs58-170](<https://github.com/zed-industries/zed/blob/4109c9dd/crates/workspace/src/searchable.rs#L58-L170>) [crates/search/src/buffer_search.rs98-500](<https://github.com/zed-industries/zed/blob/4109c9dd/crates/search/src/buffer_search.rs#L98-L500>)

### ProjectItem - File-Backed Content

The `ProjectItem` trait (defined in the `project` crate) represents items backed by project files. The `ProjectItemRegistry` in the workspace manages how these are opened and wrapped in workspace items.

**Sources:** [crates/workspace/src/workspace.rs623-749](<https://github.com/zed-industries/zed/blob/4109c9dd/crates/workspace/src/workspace.rs#L623-L749>)

## ProjectItem Trait

The `ProjectItem` trait (from the `project` crate) represents items backed by project files. The workspace's `ProjectItemRegistry` manages how these are opened and displayed.

### ProjectItem Integration


**Sources:** [crates/workspace/src/workspace.rs612-749](<https://github.com/zed-industries/zed/blob/4109c9dd/crates/workspace/src/workspace.rs#L612-L749>) [crates/workspace/src/item.rs714-757](<https://github.com/zed-industries/zed/blob/4109c9dd/crates/workspace/src/item.rs#L714-L757>)

### Opening Files

When a file is opened:

  1. The workspace calls `ProjectItemRegistry::open_path()`
  2. Each registered `try_open` function is tried in reverse registration order
  3. The first successful `try_open` returns a project item
  4. The corresponding `for_project_item` function wraps it in a workspace item
  5. The item is added to the active pane


**Sources:** [crates/workspace/src/workspace.rs706-722](<https://github.com/zed-industries/zed/blob/4109c9dd/crates/workspace/src/workspace.rs#L706-L722>)

## Opening Items

Items are opened through several pathways, all coordinated by the workspace and pane system.

### Opening Files - ProjectItemRegistry Flow

**Diagram: File Opening Flow**


The `ProjectItemRegistry` maintains two types of registered functions:

  1. `build_project_item_for_path_fns`: Functions that attempt to open a path as a specific project item type
  2. `build_project_item_fns_by_type`: Functions that wrap a project item in a workspace item


When opening a path, the registry iterates through registered types in reverse order (last registered tries first), allowing extensions to override default behavior.

**Sources:** [crates/workspace/src/workspace.rs623-749](<https://github.com/zed-industries/zed/blob/4109c9dd/crates/workspace/src/workspace.rs#L623-L749>) [crates/workspace/src/workspace.rs706-734](<https://github.com/zed-industries/zed/blob/4109c9dd/crates/workspace/src/workspace.rs#L706-L734>)

### Item Addition to Panes

**Diagram: Pane::add_item() Process**


Panes store items in `items: Vec<Box<dyn ItemHandle>>`, with `active_item_index` tracking which is visible. Preview tabs (temporary tabs) can be replaced by new preview items until the user explicitly converts them to permanent tabs.

**Sources:** [crates/workspace/src/pane.rs1402-1569](<https://github.com/zed-industries/zed/blob/4109c9dd/crates/workspace/src/pane.rs#L1402-L1569>)

## Closing Items

Item closure involves save prompts, event cleanup, and potential navigation history updates.

### Close Flow

**Diagram: Pane::close_items() Process**


The `SaveIntent` enum controls whether to prompt the user:

  * `SaveIntent::Close`: Prompt if dirty
  * `SaveIntent::Skip`: Never save, just close
  * `SaveIntent::Overwrite`: Save dirty items without prompting


**Sources:** [crates/workspace/src/pane.rs2461-2670](<https://github.com/zed-industries/zed/blob/4109c9dd/crates/workspace/src/pane.rs#L2461-L2670>) [crates/workspace/src/pane.rs80-99](<https://github.com/zed-industries/zed/blob/4109c9dd/crates/workspace/src/pane.rs#L80-L99>)

## Item Activation and Deactivation

Items transition between active and inactive states as the user switches tabs.

### Activation Flow

**Diagram: Item Activation Sequence**


When an item is activated:

  1. The previously active item (if any) has `deactivated()` called
  2. The pane's `active_item_index` is updated
  3. The item is added to `activation_history` with a timestamp
  4. An `Event::ActivateItem` is emitted
  5. The item is focused if requested
  6. The toolbar updates to show the new item's toolbar items


**Sources:** [crates/workspace/src/pane.rs887-976](<https://github.com/zed-industries/zed/blob/4109c9dd/crates/workspace/src/pane.rs#L887-L976>)

### Deactivation

When an item is deactivated (tab loses focus), `item.deactivated()` is called. Items can use this to:

  * Stop timers or polling
  * Release resources
  * Save transient state


**Sources:** [crates/workspace/src/item.rs210](<https://github.com/zed-industries/zed/blob/4109c9dd/crates/workspace/src/item.rs#L210-L210>) [crates/workspace/src/pane.rs917](<https://github.com/zed-industries/zed/blob/4109c9dd/crates/workspace/src/pane.rs#L917-L917>)

## Summary

The Item and Panel systems provide Zed's foundation for displaying diverse content types in a consistent, flexible manner:

  * **Item Trait** : Base interface for all pane content (editors, search results, AI threads)
  * **Specialized Traits** : SerializableItem, FollowableItem, SearchableItem extend functionality
  * **Type Erasure** : ItemHandle enables heterogeneous collections and dynamic dispatch
  * **Panels** : Fixed-position components in workspace docks
  * **Lifecycle** : Well-defined lifecycle with events for integration
  * **Registration** : Global registries enable discovery and construction of items


This architecture allows new item types to be added without modifying core workspace code, while providing rich integration with features like collaboration, persistence, and navigation.
