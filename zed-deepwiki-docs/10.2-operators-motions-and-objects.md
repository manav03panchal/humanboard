## Purpose and Scope

This document describes the composable command grammar that forms the core of Vim's editing model in Zed. It covers how operators (actions like delete, change, yank) combine with motions (movements like word, line, paragraph) and text objects (structured regions like words, sentences, quoted text) to create powerful editing commands.

For information about the overall mode system and state transitions, see [Mode State Machine](</zed-industries/zed/10.1-mode-state-machine>). For visual selection behavior, see [Visual Mode](</zed-industries/zed/10.3-visual-mode>).

## The Composable Grammar

Vim's editing commands follow a composable grammar where operators combine with motions or text objects to specify what action to perform on what text:
    
    
    [count] operator [count] motion/object
    

Examples:

  * `d2w` \- delete two words
  * `ci"` \- change inside quotes
  * `y$` \- yank to end of line
  * `3dd` \- delete three lines
  * `gUiw` \- uppercase inner word

## Operators

Operators specify what action to perform on text. When an operator is triggered, it is pushed onto an operator stack and Vim enters operator-pending mode, waiting for a motion or object to complete the command.

### Operator Types

Operator| Vim Key| Action Name| Effect  
---|---|---|---  
`Change`| `c`| `PushChange`| Delete text and enter insert mode  
`Delete`| `d`| `PushDelete`| Delete text  
`Yank`| `y`| `PushYank`| Copy text  
`Indent`| `>`| `PushIndent`| Increase indentation  
`Outdent`| `<`| `PushOutdent`| Decrease indentation  
`Lowercase`| `gu`| `PushLowercase`| Convert to lowercase  
`Uppercase`| `gU`| `PushUppercase`| Convert to uppercase  
`OppositeCase`| `g~`| `PushOppositeCase`| Toggle case  
`ToggleComments`| `gc`| `PushToggleComments`| Toggle line comments  
`AutoIndent`| `=`| `PushAutoIndent`| Auto-indent lines  
`Replace`| `r`| `PushReplace`| Replace characters  
`AddSurrounds`| `ys`| `PushAddSurrounds`| Add surrounding delimiters  
`ChangeSurrounds`| `cs`| `PushChangeSurrounds`| Change surrounding delimiters  
`DeleteSurrounds`| `ds`| `PushDeleteSurrounds`| Delete surrounding delimiters

### Inner vs Around

Text objects support two modes controlled by the `i` (inner) and `a` (around) operators:

Modifier| Key| Behavior| Example  
---|---|---|---  
Inner| `i`| Selects content without delimiters| `ciw` \- change inside word (no spaces)  
Around| `a`| Selects content with delimiters| `caw` \- change around word (includes trailing space)  
  
**Comparison Table**

Command| Text: `foo (bar baz) qux`| Selection with cursor on `bar`  
---|---|---  
`viw`| Inner word| `bar`  
`vaw`| Around word| `bar `  
`vi(`| Inner parens| `bar baz`  
`va(`| Around parens| `(bar baz)`  
  
### Object Selection Algorithm

**Delimiter-Based Object Selection**


The `cover_or_next` function implements this algorithm:

  1. **Covering pairs** : Find all delimiter pairs on the current line that contain the cursor 
     * If found, select the smallest (narrowest) pair
  2. **Next pairs** : If no covering pairs, find the next pair on the line after the cursor
  3. **Fallback** : If neither found, use `innermost_enclosing_bracket_ranges` to search outward


For the `inner` modifier:

  * Start after opening delimiter
  * End before closing delimiter


For the `around` modifier:

  * Include opening delimiter
  * Include closing delimiter

### Operator Implementations

Each operator has a dedicated implementation function:

**Delete Motion**


Flow:

  1. Expand selections using `motion.expand_selection()`
  2. Determine `MotionKind` (Linewise, Exclusive, Inclusive)
  3. Copy ranges to register via `copy_ranges()`
  4. Delete text with `editor.insert("")`
  5. Adjust cursor positions based on motion kind
  6. Clip cursor to valid positions


Sources: [crates/vim/src/normal/delete.rs17-90](<https://github.com/zed-industries/zed/blob/4109c9dd/crates/vim/src/normal/delete.rs#L17-L90>)

Flow:

  1. Expand selections (with special handling for word motions)
  2. Delete text
  3. Switch to Insert mode
  4. Position cursor appropriately for insertion

## Summary

The operator-motion-object system implements Vim's composable editing grammar through:

  1. **Operator Stack** \- Manages pending operators waiting for motions/objects
  2. **Motion Expansion** \- Calculates text ranges based on movement commands
  3. **Object Selection** \- Identifies semantic text regions using delimiters or syntax
  4. **Execution Pipeline** \- Applies operators to expanded selections with appropriate register handling


This architecture enables the powerful command combinations that make Vim editing efficient, while maintaining clear separation between the three concerns: what action (operator), what movement (motion), and what structure (object).
